## Entity: Product

Represents a catalog item owned exclusively by this service. No other service
queries the Product Service's database directly — they interact only through the
REST API.

### Fields

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| `_id` | ObjectId | Auto-generated by MongoDB | Exposed as `id` (string) in API responses |
| `name` | String | Required; 1–200 characters; trimmed | Display name of the product |
| `description` | String | Optional; max 2000 characters; trimmed | Detailed product description |
| `price` | Number | Required; > 0 (min 0.01); decimal | Unit selling price |
| `category` | String | Required; 1–100 characters; trimmed | Free-form category label (no separate entity in v1) |
| `stock` | Number | Required; integer; ≥ 0 | Current inventory quantity |
| `createdAt` | Date | Auto-set on insert (Mongoose `timestamps`) | ISO 8601 timestamp |
| `updatedAt` | Date | Auto-updated on every save (Mongoose `timestamps`) | ISO 8601 timestamp |

### Validation Rules

- `name`: non-empty string, max 200 characters, trimmed of leading/trailing whitespace.
- `description`: optional; if provided, max 2000 characters, trimmed.
- `price`: positive number (> 0); enforced at both zod schema layer and Mongoose `min: 0.01`.
- `category`: non-empty string, max 100 characters, trimmed.
- `stock`: non-negative integer (≥ 0); enforced at both zod schema layer (`z.number().int().min(0)`) and Mongoose `min: 0`.
- Unknown fields in request body are silently stripped (zod `.strip()` default).
- All `PATCH` updates run Mongoose validators (`runValidators: true`) to enforce schema constraints on partial writes.

### State Transitions

Products have no explicit status field in v1. The only lifecycle events are:

```
[Not Exists] ──── POST /api/products ────────► [Active]
[Active]     ──── PATCH /api/products/:id ───► [Active] (fields updated)
[Active]     ──── DELETE /api/products/:id ──► [Not Exists] (hard delete)
```

Soft-delete is out of scope for v1 (documented in spec Assumptions).

### Indexes

| Index | Fields | Type | Purpose |
|-------|--------|------|---------|
| Primary | `_id` | Unique (default) | Document identity |
| List query | `createdAt` | Ascending (compound candidate) | Consistent ordering for paginated list; supports `sort: { createdAt: -1 }` for newest-first default |

> **Note**: A category index is deferred to a future iteration when filtering by category
> is introduced. At ≤ 10,000 documents a collection scan is within SLO.

### Mongoose Schema (TypeScript representation)

```typescript
// Authoritative shape — implementation must match exactly.
interface IProduct {
  _id: Types.ObjectId;
  name: string;
  description?: string;
  price: number;          // stored as Number; > 0
  category: string;
  stock: number;          // stored as Number; integer; >= 0
  createdAt: Date;
  updatedAt: Date;
}
```

The Mongoose schema MUST include a `toJSON` transform that:
1. Renames `_id` → `id` (string).
2. Removes `__v` (Mongoose version key).
3. Does NOT strip any other field (no sensitive data on Product unlike User's `passwordHash`).

### Public API Representation

Returned in all CRUD responses:

```json
{
  "id": "664f1a2b3c4d5e6f7a8b9c0d",
  "name": "Wireless Keyboard",
  "description": "Compact 75% layout, RGB backlit",
  "price": 79.99,
  "category": "Electronics",
  "stock": 150,
  "createdAt": "2026-05-27T10:00:00.000Z",
  "updatedAt": "2026-05-27T10:00:00.000Z"
}
```

### Paginated List Response Envelope

`GET /api/products` returns:

```json
{
  "data": [ /* array of product objects */ ],
  "total": 420,
  "page": 1,
  "limit": 20,
  "totalPages": 21
}
```

## Zod Validation Schemas

### Create Product Body (`POST /api/products`)

```typescript
const CreateProductSchema = z.object({
  name:        z.string().trim().min(1).max(200),
  description: z.string().trim().max(2000).optional(),
  price:       z.number().positive(),
  category:    z.string().trim().min(1).max(100),
  stock:       z.number().int().min(0),
});
```

### Update Product Body (`PATCH /api/products/:id`)

```typescript
const UpdateProductSchema = z.object({
  name:        z.string().trim().min(1).max(200).optional(),
  description: z.string().trim().max(2000).optional(),
  price:       z.number().positive().optional(),
  category:    z.string().trim().min(1).max(100).optional(),
  stock:       z.number().int().min(0).optional(),
}).refine(obj => Object.keys(obj).length > 0, {
  message: 'At least one field must be provided for update',
});
```

### Pagination Query Params (`GET /api/products`)

```typescript
const ListQuerySchema = z.object({
  page:  z.coerce.number().int().min(1).default(1),
  limit: z.coerce.number().int().min(1).max(100).default(20),
});
```

### ID Path Param (all `:id` routes)

```typescript
const IdParamSchema = z.object({
  id: z.string().regex(/^[a-f\d]{24}$/i, 'Invalid product ID'),
});
```
